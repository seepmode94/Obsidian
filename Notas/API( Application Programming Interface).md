Uma API é uma interface que permite a comunicação entre aplicações, sistemas e dispositivos, funcionando através de um ciclo de pedido e resposta. O artigo apresenta os elementos básicos de uma chamada API, como endpoint, método HTTP, parâmetros, headers, corpo do pedido e resposta com códigos de estado. Também usa a metáfora do restaurante: o cliente faz o pedido, o empregado transmite e a cozinha executa.

Depois, o texto resume porque as APIs são importantes: ajudam na automação, aceleram a inovação, melhoram a segurança e reduzem custos ao permitir reutilizar serviços já existentes. Distingue ainda vários tipos de API, como private, public e partner, e descreve estilos comuns como REST, SOAP, GraphQL, Webhooks e gRPC.

Por fim, mostra casos de uso frequentes, como integração entre sistemas, adicionar funcionalidades a apps, ligar dispositivos IoT, suportar microservices e reforçar governação e segurança. Também refere exemplos reais de empresas que expõem APIs, como Salesforce, Notion, Discord, Pinterest e DoorDash.
